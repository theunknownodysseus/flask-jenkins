Application Deployment Automation using Jenkins

Application deployment currently requires manual intervention, causing delays and human errors. There is a need to automate the entire deployment pipeline using Jenkins to ensure consistency, speed, and reliability of application releases.

AIM

To configure Jenkins jobs (freestyle and pipeline) to automate application deployment, enabling automatic deployment after every successful build and monitoring the deployment status through console logs and build history.

PROCEDURE

Overall Flow

[ Code Commit / Manual Trigger ]
        ↓
[ Jenkins Job (Freestyle / Pipeline) ]
        ↓
[ Build & Test Stage ]
        ↓
[ Docker Build & Run (Deployment) ]
        ↓
[ Monitor via Console Output / Build History ]

Step 1: Set Up Jenkins

Install Jenkins on Ubuntu VM (IP: 192.168.2.129)

sudo apt update && sudo apt install -y openjdk-17-jdk
wget -q -O - https://pkg.jenkins.io/debian-stable/jenkins.io-2023.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt update && sudo apt install -y jenkins
sudo systemctl start jenkins && sudo systemctl enable jenkins

Access Jenkins: http://192.168.2.129:8080

Step 5: Create a Pipeline Job

  • Job: firstpipeline
pipeline {
    agent none
    stages {
        stage('Print Hello') {
            steps {
                sh 'pwd && git && ls -la'
            }
        }
        stage('Docker') {
            steps {
                sh 'docker build -t web134'
                sh 'docker run -d -p 5000:5000 web134'
            }
        }
    }
}
